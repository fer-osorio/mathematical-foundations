üîê What Makes RSA Secure?

RSA security relies on the integer factorization problem:

  • Easy: Multiply two large primes: p √ó q = n (milliseconds)
  • Hard: Factor n back into p and q (years to millennia for large n)

If an attacker could factor n efficiently, they could:

  1. Recover p and q from n
  2. Compute œÜ(n) = (p-1)(q-1)
  3. Compute private key d = e-1 mod œÜ(n)
  4. Decrypt all messages!

This tool demonstrates "textbook RSA" for educational purposes. It has several vulnerabilities:

1. No Padding (Critical)

Without padding schemes like OAEP:

  • Deterministic: Same message always produces same ciphertext (attacker can recognize patterns)
  • Malleable: Attacker can manipulate ciphertext: c' = c √ó 2e mod n decrypts to 2m
  • Small message space: Attacker can precompute encryptions of common messages

2. Client-Side Key Storage

Keys are stored in browser memory:

  • Vulnerable to XSS attacks
  • Visible in browser DevTools
  • Not protected by hardware

Production solution: Use Hardware Security Modules (HSMs) or key management services

3. Timing Attacks

Modular exponentiation time depends on private exponent d:

  • Number of squaring operations reveals bits of d
  • JavaScript timing is observable via Performance API

Mitigation: Use constant-time algorithms (Montgomery ladder)

4. Small Key Sizes

512-bit and 1024-bit RSA are INSECURE for real-world use:

  • 512-bit RSA: Broken in 1999
  • 1024-bit RSA: Deprecated by NIST since 2013
  • 2048-bit RSA: Current minimum standard
  • 3072-bit+ RSA: Recommended for long-term security

‚úÖ Real-World RSA Best Practices

  • Use OAEP padding (Optimal Asymmetric Encryption Padding)
  • Minimum 2048-bit keys (3072-bit for high security)
  • Store keys in HSMs (Hardware Security Modules)
  • Use hybrid encryption (RSA for key exchange, AES for data)
  • Implement proper key rotation (change keys periodically)
  • Use standard libraries (OpenSSL, libsodium - not hand-rolled crypto)

üìö Further Learning

To understand RSA security deeply, study:

  • Number Theory: Modular arithmetic, Euler's theorem, Chinese Remainder Theorem
  • Computational Complexity: P vs NP, reduction proofs, hardness assumptions
  • Cryptanalysis: Factorization algorithms (Quadratic Sieve, Number Field Sieve)
  • Side-Channel Attacks: Timing, power analysis, fault injection
  • Post-Quantum Cryptography: Lattice-based, code-based alternatives (RSA vulnerable to quantum computers)
